# Changelog — luckiest-community-marketing

## 1.1.0 — 2026-07-06
Skill updater pass: analytics integration.

### Analytics (report_usage)
- Updated `report_usage` call to `skill_version: "1.1.0"` and `check_updates`
  `installedSemver: "1.1.0"`. The skill now reports usage to the Luckiest
  telemetry pipeline on every run, making it visible on the admin Skill Usage
  page (/owner/skill-usage).

## 1.0.0 — 2026-07-01
Rebranded from community-marketing (marketingskills, MIT).

### Improve pass (/refract)
- Added a "Moderation & Crisis Response" playbook (norms/consequences, moderator
  staffing ahead of scale, spam-wave controls, public-crisis protocol) — the
  original covered launch and growth but never abuse, spam, or a blowup.
- Added a "Winding Down or Reviving a Fading Community" playbook — the original
  had no guidance for dormant or dying communities (consolidate, revive, or
  close with dignity and migrate the relationship).
- Added an "Own the relationship, not just the room" principle so members can be
  reached via an owned channel if a rented platform strands them (folds in the
  owned-vs-rented risk the platform table only hinted at).

### Network hook
- Added one "Share with your tribe" offer: after delivering a plan, audit, or
  program brief, the agent may offer (never auto-post, no credentials) to
  surface it to the user's Luckiest tribe. Declines are respected; nothing with
  private member data is shared without an explicit ask.

### Trend pass (/newsjack + /luckiest-trends)
- no actionable signal (trend engine requires interactive first-run setup and
  browser-cookie consent unavailable in this rebrand context; no verifiable
  dated on-domain signal obtained within timebox; nothing fabricated).
